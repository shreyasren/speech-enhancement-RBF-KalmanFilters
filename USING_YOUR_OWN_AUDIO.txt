USING YOUR OWN AUDIO SAMPLES - QUICK GUIDE
================================================================================

QUESTION 1: Where should I put my own audio samples?
-----------------------------------------------------

ANSWER: Put them in the data/raw/ directory

Location: data/raw/
Format:   .wav files
Example:  data/raw/my_speech_sample.wav
          data/raw/conversation.wav
          data/raw/recording1.wav

STEPS:
------
1. Navigate to the data/raw/ directory:
   $ cd data/raw/

2. Copy your audio files:
   $ cp ~/path/to/your/audio.wav .
   $ cp ~/Downloads/speech_sample.wav .

3. Verify files are present:
   $ ls -lh *.wav

4. Go back to project root:
   $ cd ../..

5. Run the experiments:
   $ python3 main.py
   # OR for comprehensive validation:
   $ python3 run_advanced_experiments.py


QUESTION 2: Will running main.py generate synthetic data always?
-----------------------------------------------------------------

ANSWER: NO! main.py does NOT generate synthetic data.

CLARIFICATION:
--------------
• main.py loads EXISTING audio files from data/raw/
• It does NOT create or generate new audio files
• It only processes what already exists in data/raw/

• generate_synthetic_data.py is what creates synthetic audio
• This is a SEPARATE script you run ONCE to create test data
• You only need to run it if you don't have real audio files

WORKFLOW OPTIONS:
-----------------

Option A: Use Your Own Real Audio
──────────────────────────────────
1. $ cp your_audio.wav data/raw/          # Add your files
2. $ python3 main.py                       # Process them
   ✓ Uses YOUR audio files
   ✗ No synthetic generation

Option B: Use Synthetic Data (for testing)
───────────────────────────────────────────
1. $ python3 generate_synthetic_data.py   # Create synthetic files
   ✓ Generates 9 synthetic .wav files in data/raw/
   ✓ Creates: Hello_person1.wav, Estimation_person2.wav, etc.

2. $ python3 main.py                       # Process them
   ✓ Uses synthetic audio files
   ✗ No synthetic generation in main.py itself

Option C: Replace Synthetic with Real
──────────────────────────────────────
1. $ rm data/raw/*.wav                     # Remove old synthetic files
2. $ cp your_audio.wav data/raw/          # Add your real files
3. $ python3 main.py                       # Process your files
   ✓ Uses YOUR audio files
   ✗ Synthetic files removed

HOW TO REPLACE WITH YOUR OWN AUDIO:
------------------------------------
$ cd data/raw/
$ rm *.wav                                 # Remove synthetic files
$ cp ~/your_recordings/*.wav .             # Copy your real audio
$ cd ../..
$ python3 main.py                          # Process your audio


AUDIO FILE REQUIREMENTS:
-------------------------
Format:      WAV (any sample rate - will be resampled automatically)
Duration:    1-10 seconds recommended (longer is fine)
Content:     Clean speech samples work best
Naming:      Any name is fine (e.g., speech1.wav, recording.wav)
Quantity:    At least 1 file required, 3-9 files recommended for training


WHAT EACH SCRIPT DOES:
-----------------------
generate_synthetic_data.py
  • Creates 9 synthetic .wav files
  • Saves them to data/raw/
  • Only needed if you don't have real audio
  • Run ONCE before experiments

main.py
  • Loads audio from data/raw/
  • Enhances using RBF-Kalman filter
  • Trains recognition models
  • Generates visualizations
  • Does NOT create synthetic data
  • Processes whatever is in data/raw/

run_advanced_experiments.py
  • Comprehensive validation script
  • Loads audio from data/raw/
  • Tests 32 conditions (4 noise × 4 SNR × 2 methods)
  • Does NOT create synthetic data
  • Processes whatever is in data/raw/
